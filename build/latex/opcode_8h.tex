\begin{DoxyItemize}
\item 从 dram 中读取的用于执行的指令是 32 位宽的。 这 32 位包含所有要执行的操作对应的指令， 比如操作码，用于读/写的源寄存器和目标寄存器， 立即数值等等。
\item 对于不同类型的指令的译码是不同的，指令类型有以下几种：
\end{DoxyItemize}
\begin{DoxyEnumerate}
\item R 类型：寄存器类型指令
\item I 类型：立即数类型指令
\item S 类型：存储类型指令
\item B 类型：中断类型指令
\item U 类型：寄存器类型指令
\item J 类型：跳转类型指令
\end{DoxyEnumerate}

\begin{DoxyEnumerate}
\item {\ttfamily opcode}： 低 7 位（inst\mbox{[}6\+:0\mbox{]}）用于区别每个 CPU 操作。
\item {\ttfamily rd}：一个 4 位值（inst\mbox{[}11\+:7\mbox{]}），给定目标寄存器的地址。
\item {\ttfamily funct3}：一个 3 位值（inst\mbox{[}14\+:12\mbox{]}）， 每一组类似操作的操作码相同（如 ADD，\+SUB，\+DIV，\+MUL 等等）。 每组可包括至多 8 个不同指令，对应 8 个不同的{\ttfamily funct3}值。
\item {\ttfamily funct7}：一个 7 位值（inst\mbox{[}31\+:25\mbox{]}）。与{\ttfamily funct3}类似， {\ttfamily funct7}将一组相同的{\ttfamily funct3}指令划分为多个指令。 例如，\+SR（右移）有两个指令： {\ttfamily SRA}（算数右移）与{\ttfamily SRL}（逻辑右移）对应不同的{\ttfamily funct7}。
\item {\ttfamily rs1}：一个 4 位值（inst\mbox{[}19\+:15\mbox{]}），给定源寄存器 1 的地址。
\item {\ttfamily rs2}：一个 4 位值（inst\mbox{[}24\+:20\mbox{]}），给定源寄存器 2 的地址。
\item {\ttfamily imm}：表示立即数。
\item {\ttfamily shamt}：shamt 位于{\ttfamily imm}的低位，存有移位指令的移位量。 
\end{DoxyEnumerate}
